Run a command within a container

Containers are related to virtual machines, but aren‚Äôt the same thing. The main difference between a container and a virtual machine is that a container does not contain or run an entire operating system. Instead, containers run within an existing operating system. The theory is that a container uses fewer resources because it doesn‚Äôt carry all the extra baggage of a complete operating system.

One fairly common use of containers is to distribute complex software and its dependencies as a single ‚Äúimage‚Äù, similar to what you downloaded for SerenityOS when you were setting up a virtual machine. An image is basically an archive of an operating system‚Äôs folder structure.

We could spend an entire course talking about the technical foundations of containers (that would be an amazing systems course), but in this course our goal is simple: install enough software to get a command running in a container.

Undoubtedly the most popular container management software is Docker.

Docker has a highly polished visual environment for managing containers called Docker Desktop.

If you‚Äôre installing Docker Desktop on Windows, you‚Äôre going to need to install the Windows Subsystem for Linux. There‚Äôs a link in Docker‚Äôs documentation, but for completeness, you can find documentation about how to install WSL on Microsoft‚Äôs web site.

You should use this file to:

There are a lot of options in this, but most of them are actually options for pandoc and not Docker. Let‚Äôs step through the Docker options:

That‚Äôs it for Docker options! The rest are options to pandoc that you can read about in the documentation for pandoc.
